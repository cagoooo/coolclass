# 🔮 未來優化建議（依優先順序）

## 🥇 第一優先：體驗微調（低風險 · 高 ROI）

這層是「現在使用者已經在用、再優化一點就大進步」的範疇。

### A1. Onboarding 升級到 v3.7 版本（**2 小時**）

**為什麼**：onboarding.js 還停在 v3.0 的「8 工具介紹」，使用者裝上 App 第一次看到的引導，跟現在 26 工具有落差。新老師會被導到只認識 8 個工具。

**怎麼做**：
- 改 5 卡為 6 卡：第 6 卡介紹「右下角 🧭 工具導航 + ⬆ 回頂部」
- 第 3 卡（介紹工具）改成「分類速覽」，列 6 大組
- 加「想看 26 工具完整清單？點 🧭」CTA

**驗收**：第一次打開 → 引導完 → 知道有 26 工具、知道哪裡找導航。

---

### A2. 首頁「我的常用工具」置頂（**3 小時**）

**為什麼**：26 工具排在首頁要捲很長。每個老師常用的其實只有 8-10 個。

**怎麼做**：
- 每個工具卡右上角加 ⭐ 收藏按鈕（localStorage `akai_favs_v1`）
- 首頁開頭加「⭐ 我的常用」橫排顯示已收藏的（最多 8 個）
- 沒收藏任何工具時顯示「點工具卡的 ⭐ 來收藏」提示

**驗收**：第二次來首頁直接看到常用工具，不用捲。

---

### A3. 跑馬燈訊息語音播報（**1 小時**）

**為什麼**：能量卡有語音播報、抽號有語音播報，跑馬燈卻沒有。當投影 + 投影機沒擴音時，老師唸出來費力。

**怎麼做**：
- 在跑馬燈設定區加「🔊 語音播報」開關
- 每則訊息開頭時 `speechSynthesis.speak(new SpeechSynthesisUtterance(msg))`
- 加語速、音量、選擇中文女聲/男聲

**驗收**：訊息滑過時語音同步唸出。

---

### A4. 鍵盤快捷鍵總覽頁（**1 小時**）

**為什麼**：抽號 / 計時器 / 跑馬燈各有快捷鍵但分散，README 寫的也少。加一個 `?` 鍵彈出 cheatsheet 對重度使用者很實用。

**怎麼做**：
- 寫 `shortcuts.js`（類似 nav.js 全頁注入）
- 按 `?` 時彈一張 modal 列出當頁支援的快捷鍵
- 跨頁通用（Esc 關閉、F 全螢幕…）也列出

---

### A5. 主題色再 +3 組（**0.5 小時**）

**為什麼**：tweaks 目前 5 色（彩虹 / 薄荷 / 日落 / 莓果 / 海洋），但老師可能想要：
- 🎄 聖誕節主題（綠 + 紅 + 金）
- 🌸 春節 / 元宵（紅 + 金 + 黃）
- 🌳 大自然 / 環保日（綠 + 棕）

**驗收**：節慶月份可一鍵換主題。

## 🥈 第二優先：防呆與穩定性（資料安全）

老師最怕「資料消失」。這層不會增加新功能，但讓既有功能更穩。

### B1. 全資料移轉到 IndexedDB（**6 小時 · 中難度**）

**為什麼**：所有資料都在 localStorage，**單一 domain 上限 5MB**。隨著用一年累積（一年 200 天 × 30 人情緒 + 出勤 + 行為觀察 + 親師溝通 ...），有跑滿風險。一旦寫入失敗，使用者完全不知道。

**怎麼做**：
- 寫 `storage-v2.js`：對外保留 `getItem` / `setItem` API
- 內部根據 key 自動路由：
  - 小資料（roster, tweaks, marquee）→ localStorage（讀取最快）
  - 大資料（emotion records, attendance sheets, behavior logs, contact）→ IndexedDB
- 第一次升級時自動遷移
- 加 storage quota 偵測 + 警告

**驗收**：可以塞滿 5MB+ 資料而不壞，gallery 已經這樣做。

---

### B2. 還原時的版本相容檢查（**2 小時**）

**為什麼**：v3.5 → v3.6 → v3.7 資料結構有微調。如果使用者下載舊版備份，匯入到新版可能 partial 失敗。目前沒檢查。

**怎麼做**：
- backup.html 匯出時在 JSON 加 `__schema_version: "3.7"`
- 還原時比對版本，遇到舊版本：
  - 顯示提示「這份備份來自 v3.5，部分欄位可能需要轉換」
  - 內建 migrator function 自動補欄位
- 還原前先預覽：「將還原 23 個工具的資料，包含 X 位學生」

---

### B3. 缺資料情境的友善引導（**2 小時**）

**為什麼**：第一次用 emotion.html 但沒設名單會看到「30 個無名號碼」很尷尬。第一次用 birthday.html 沒輸入生日會看到空月曆。

**怎麼做**：
- 每個依賴 roster 的工具，在資料為空時顯示一張「空狀態卡」：
  - 圖示 + 一句話解釋為什麼空
  - 一鍵 CTA 跳到設定頁
- 例：「👥 還沒設定班級名單？點這裡 3 步驟搞定」

---

### B4. 跨工具資料一致性檢查（**3 小時**）

**為什麼**：roster.count 從 30 改成 25 時，舊有 26-30 號的情緒記錄、積點、行為觀察會變孤兒。

**怎麼做**：
- 在 roster.html 改人數時偵測：「有 5 位學生 (26-30 號) 在 X 個工具有資料，是否一併刪除 / 保留？」
- 寫 `data-doctor.js` 在 backup.html 加「🩺 健康檢查」按鈕，掃出孤兒資料

## 🥉 第三優先：教學現場新需求（新工具）

26 工具還有缺，下面是真實老師需求清單。

### C1. 📖 閱讀紀錄表（**6 小時**）

**為什麼**：「閱讀小博士」是國小普遍活動，老師要追蹤每個學生本學期讀幾本書、寫幾篇心得。

**怎麼做**：
- 每位學生獨立紀錄：書名、作者、心得字數、日期、教師評語
- 月度排行：本月讀最多的前 5 名
- 學期結算：印「閱讀獎狀」A4

---

### C2. 📝 學習單產生器（**8 小時**）

**為什麼**：老師要出隨堂測驗、家庭作業，現有方式（Word / Google 表單）切換麻煩。

**怎麼做**：
- 4 種題型：選擇 / 填空 / 簡答 / 連連看
- 拖曳排版 + 即時預覽
- 一鍵印 A4，可隱藏答案版 / 含答案版
- 題庫可重複用，按主題分類
- 後續可接 Gemini API 自動生題（見 E1）

---

### C3. 🏃 體適能 / 運動會成績登錄（**4 小時**）

**為什麼**：每學期教育部要求填報的體適能（仰臥起坐、坐姿體前彎、立定跳遠、800/1600 米）目前要老師手動 Excel。

**怎麼做**：
- 5 個項目 × N 學生網格
- 自動算 PR 值（百分等級）+ 達成等級（金/銀/銅/未通過）
- 匯出 CSV 直接吃進教育部表格

---

### C4. 🎵 節拍器 + 律動計時（**2 小時**）

**為什麼**：音樂課、健康操、午休時段用得到。

**怎麼做**：
- 節拍器：可調 BPM、4/4 拍 / 3/4 拍、視覺 + 聽覺雙重提示
- 律動計時：跟著音樂節奏間隔提示「換動作」

---

### C5. 🌱 環保 / 蔬食日打卡（**3 小時**）

**為什麼**：很多學校推週一蔬食、垃圾減量。

**怎麼做**：類似 emotion 的打卡，但項目是「✅ 自帶餐具 / ✅ 蔬食午餐 / ✅ 不剩飯」三項。

---

### C6. 🎤 課堂錄音 → 文字稿（**8 小時 · 高潛力**）

**為什麼**：老師家訪、議題討論需要保留紀錄，事後整理超累。

**怎麼做**：
- Web Speech API 即時辨識（中文）
- 自動分段、加時間戳
- 可選擇「逐字稿」或「重點摘要」（若有接 Gemini）
- 匯出純文字、Markdown、PDF

## 🚀 第四優先：推廣與生態（讓更多老師看到）

工具做好但沒人用就可惜了。

### D1. 教師示範影片 + 教學頁（**4 小時**）

**為什麼**：老師看到 26 個工具會 overwhelm，需要「為什麼這個對你有用」的故事。

**怎麼做**：
- 寫一個 `tutorial.html`：每個工具一個 30 秒 GIF + 一段使用情境 + 「立刻試試」CTA
- 可用 OBS 錄螢幕 → 用 gif-converter 轉成檔
- 上 Youtube 一條 5 分鐘總覽影片

---

### D2. 教師回饋表（**2 小時**）

**為什麼**：不知道使用者用了什麼工具、卡在哪、缺什麼。盲推一年也不知道方向。

**怎麼做**：
- 在每個工具頁腳加「💬 給阿凱老師建議」連結
- 連到一個 Google Form（不會洩漏老師個資）
- 或用 Formspree / Netlify Form 自架

---

### D3. 匿名遙測（**3 小時 · 可選**）

**為什麼**：知道哪些工具熱門、哪些幾乎沒人開，可以聚焦優化。

**怎麼做**：
- 用 Plausible / Umami / GoatCounter 任一個免費的隱私友善 analytics
- 只記 pageview，不記任何個人資料
- 在頁腳寫明「我們有用 X 統計訪問次數，不會收集任何學生資料」

---

### D4. LINE 分享深連結（**2 小時**）

**為什麼**：老師在 LINE 群裡常常會貼工具截圖 + URL，但 URL 都是首頁。應該能 deep link 到特定工具甚至預填內容。

**怎麼做**：
- 每個工具頁加「📤 分享」按鈕
- 產生 LINE share URL（含 OG meta tags 預覽）
- 進階：跑馬燈支援 `?msg=xxx` query string 預填訊息

---

### D5. 多語化（i18n）（**8 小時 · 中難度**）

**為什麼**：台灣國際學校、海外華語學校、東南亞華僑學校都可能想用，但介面全中文卡關。

**怎麼做**：
- 抽出所有文字到 `i18n/zh-TW.json` / `i18n/en.json`
- 加語言切換按鈕（tweaks 面板裡）
- 第一階段先做英文 + 簡體中文兩個

## 🌍 第五優先：可選的雲端整合（高難度但開新格局）

純靜態工具的天花板：跨裝置同步。如果做了會大幅吸引「在學校用桌機，回家用手機」的使用者。

### E1. ☁️ Supabase + Google OAuth 跨裝置同步（**12 小時 · 高難度**）

**為什麼**：老師最常抱怨「換電腦資料沒了」「家裡看不到學校的紀錄」。

**怎麼做**：
- 走「漸進式雲端」路線：
  - 預設仍是 localStorage（不需登入也能用）
  - 點「☁️ 啟用雲端同步」→ Google OAuth 登入 → 把 localStorage 鏡像到 Supabase
  - 換裝置登入 → 自動拉下來合併
- 衝突解決：用 `updated_at` 時間戳，後寫贏
- **重要**：必須仍可離線使用，雲端只當「冗餘備份」

**注意事項**：見現有 skill `supabase-google-oauth-integration` + `supabase-free-tier-keepalive`。

---

### E2. 📂 Google Drive 自動備份（**6 小時**）

**為什麼**：比 Supabase 簡單、不用後端、老師熟悉。

**怎麼做**：
- Google Drive API + Picker API
- 「💾 啟用 Google Drive 自動備份」→ 授權 → 每天自動上傳一份 JSON 到老師指定資料夾

---

### E3. 👨‍👩‍👧 家長 view-only 連結（**8 小時**）

**為什麼**：家長想看自家小孩的情緒紀錄、行為觀察、積點，但又不能讓家長看到全班的。

**怎麼做**：
- 在 student.html 加「📤 產生家長分享連結」
- 用 Supabase Row Level Security：只能讀 student_id = X 的資料
- 產生 magic link，家長點開無需登入直接看

## 🤖 第六優先：AI 整合（高潛力、阿凱老師熟悉）

阿凱老師專長就是教 AI，這層應該是強項。

### F1. 親師溝通 → AI 草擬回覆（**4 小時**）

**為什麼**：家長在 contact.html 留言「孩子最近狀況」後，老師要花心力擬回覆。AI 可以給草稿。

**怎麼做**：
- contact.html 加「✨ AI 草擬回覆」按鈕
- 呼叫 Gemini API（用使用者自己的 free key）
- 提示詞範本：「以小學老師專業且溫暖的口吻，回覆以下家長訊息...」
- 老師可改寫後再寄出

**注意**：見 skills `gemini-api-integration` + `gemini-free-tier-first` + `gcp-api-key-secure-create`。

---

### F2. 行為觀察 → 月度 AI 摘要（**3 小時**）

**為什麼**：月底要寫每個學生的「行為觀察月度摘要」很累。

**怎麼做**：
- report.html 加「✨ AI 摘要本月行為」
- 把當月所有 behavior 紀錄丟給 Gemini
- 生成「3 大優點 + 1 個成長空間 + 一段給家長的話」

---

### F3. 學期評語自動生成（**4 小時**）

**為什麼**：學期末寫成績單評語是老師最痛的事之一。每個學生 2-3 句、20-30 個學生 = 60-90 句個性化內容。

**怎麼做**：
- 在 student.html 加「📝 生成學期評語」
- 收集該學生本學期所有資料（情緒/出勤/積點/行為/作品數）
- AI 生成 3-5 句評語草稿
- 老師潤飾後一鍵複製到成績系統

---

### F4. 教案備忘 → AI 補完（**3 小時**）

**為什麼**：lesson.html 目前是純文字備忘，但老師寫到一半常卡關。

**怎麼做**：
- 加「✨ AI 補完教學重點」「✨ AI 建議教學活動」按鈕
- 根據已寫的主題、年級、課堂時間，給 3 個延伸活動建議

## 🔧 第七優先：技術債清理（持續性）

### G1. crayon.css 拆檔（**3 小時**）

**為什麼**：1100+ 行單檔 css，要找東西要 scroll 很久。

**怎麼做**：拆 `base.css` + `components.css` + `tools.css` + `print.css`，用 `@import` 串起來。

---

### G2. 共用 storage layer（**4 小時**）

**為什麼**：30 個 HTML 各自重複寫 `loadState() / saveState()`，命名混亂。

**怎麼做**：寫 `akai-store.js` 統一 API。

## 💡 我會避開的方向（給參考）

- ❌ **改用框架（React / Vue）**：純 vanilla 是最大優勢，改了會失去「fork 後立刻可改」的特性。
- ❌ **收費 / 訂閱**：定位是免費公益，加付費機制違背初心。
- ❌ **學生個人帳號**：兒少資料保護法很麻煩，目前 Kiosk 模式已經夠用。
- ❌ **整合 Firebase 全家桶**：靜態 + 偶爾雲端同步就夠，引入 Firebase 會讓 fork 門檻變高。
